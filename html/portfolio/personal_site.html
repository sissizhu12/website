              introduction


              Back to basics: Creating this website helped me better understand the worlds of design and code as collaborative rather than independent. I also gained a deeper understanding of the building blocks of websites, HTML and CSS, which brought me new insights as a website designer. Furthermore, it was exciting to go through a complete process of understanding the purpose of portfolio websites, brainstorming for my own, and both designing and coding my plans for this project.

              To orient myself for the long term, I initially set two main goals for myself:


              1. Code the website from scratch without using external plugins or templates
              2. Dig deep to learn about the proper setup and maintenance of websites

              how did I get started?


              First, I assembled a user persona for the HR or recruiter viewing my portfolio website.

              To understand the needs and pain points, I did secondary research through articles, videos,
              and interviews.

              Similarly, I asked myself how and what content should be organized on the website. To explore my options and see meaningful connections, I sorted content into groups.

              This exercise also made me consider what content was most and least important to showcase.

              When organizing the content into a more detailed user flow chart, I realized that I would need to make sure that users could quickly navigate and differentiate the pages to find what they're looking for.

              Because the most popular website entry points would be the home page or the portfolio page, I needed to make sure those pages especially had a visually salient and consistent navigation bar.

              This insight ultimately helped guide my initial designs.

              For visual inspiration, I drew on images from my short film “Dream” due to its vibrant and dreamy colours.

              I wanted to use the images as a theme to show my introspective personality, so I created initial wireframes in Figma that referenced the colour palette and tone.

              To optimise the website layout responsiveness, I realized two key guidelines:


              1. The layout of photos, videos, and embeds must flexibly scale with similar screen ratios

              2. Webpage navigation can’t overly rely on vertical scrolling

              Through explorative sketches, I found a solution of using different navigation bars and content layouts depending on screen size. At this point, I started building out the layout in code and trying variations.

              reflection


              Through the website development, I practiced not only design skills but key coding skills also:


            • Using Git for version control and regular commits (gh/sissizhu12/website)

            • Organizing files and structuring code with comments

            • Setup of automatic website deployment on GitHub workflows

            • HTML and CSS strategies for responsive webpages

              • In sum, this project made me appreciate the coding side of website development more. I often moved back and forth between designing and coding, giving me new insights from one role to support the other.
