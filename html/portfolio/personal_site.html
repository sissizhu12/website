              introduction


              Back to Basics: Building this website helped me better understand the worlds of design and code as collaborative rather than independent. I also gained a deeper understanding of the building blocks of websites, HTML and CSS, which brought me new insights as a website designer.

              To orient myself for the long term, I initially set two main goals for myself:


              1. Code the website from scratch without using external plugins or templates
              2. Dig deep to learn about the proper setup and maintenance of websites

              how did I get started?


              First, I assembled a user persona for my portfolio website to understand the HR or recruiter perspective on hiring. Focusing on exploring wants, needs, and pain points, I did research through articles, watching videos, and interviews.

              Similarly, I asked myself how and what content should be organized on the website. To explore my options and see meaningful connections, I sorted content into groups. This exercise also made me consider what content should live at the forefront of the page and what content is less important to showcase.

              When organizing the content into a more detailed user flow chart, I realized that I would need to make sure that users could quickly navigate and differentiate the pages to find what they're looking for. Because the most popular entry points of my website would to be the home page or the portfolio page itself, I needed to make sure the navigation was visually salient and consistent on those pages; this insight ultimately helped guide my initial designs.

              For visual inspiration, I was guided by images taken from my short film “Dream” due to its vibrant and dreamy colours. I wanted to use the images as a theme to show my introspective personality, so I created mockups in Figma that referenced its colour palette and feeling.

              To optimise responsive design, I realized two key guidelines:

              1. The layout of photos, videos, and embeds should scale well with screen size
              2. Webpage navigation shouldn’t overly rely on vertical scrolling
              Through explorative sketches and mockups, I decided to use slightly different navigation bars and content layout depending on screen size.

              Through the development of my website, I thoroughly learned some good coding practices:

            • Using Git for version control (gh/sissizhu12/website)
            • Ensuring regular and organized commits
            • Proper coding practices such as comments and file organization
            • Proper setup of automatic website deployment on GitHub workflows
            • Appropriate HTML and CSS strategies for responsive webpages
              • In sum, this experience made me appreciate the coding side of website development more. I found myself moving back and forth between designing and coding, which allowed me to use new insights from one role to support the other.
